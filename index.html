Identified Issues & Solutions
ISSUE #001 CRITICAL
Curriculum Content Downloadable via IDM / Browser Extensions
All PDF and video content in the curriculum section can be downloaded using third-party browser extensions like Internet Download Manager (IDM). There is no server-side protection preventing direct file access.
✦ RECOMMENDED FIX
Implement signed, expiring URLs for all media files. Serve files through an authenticated backend route rather than exposing direct CDN/storage URLs. Set proper Cache-Control headers. Disable public read access on cloud storage (S3/GCS bucket policy).
ISSUE #002 CRITICAL
Video Download via Chrome Inspector Panel — Attribute Bypass
Video elements have a nodownload attribute which can be removed using Chrome DevTools, enabling direct download of all video content.
✦ RECOMMENDED FIX
Never rely on frontend HTML attributes for security. Implement DRM (Digital Rights Management), use HLS/DASH streaming with token-authenticated manifests, and ensure video source URLs are short-lived signed tokens that expire within minutes.
ISSUE #003 HIGH
Missing Audit Logs on Admin Portal
The admin portal currently does not record user activity logs for teachers, students, or admins. Admin cannot monitor who did what, when, or for how long — making accountability and auditing impossible.
✦ RECOMMENDED FIX
Implement a centralized audit logging system that records: User ID, Role, Action Type, Module, Timestamp, IP Address, and Duration. Build a filterable admin dashboard to view and export logs per user/role/date range as PDF reports.
ISSUE #004 MEDIUM
Complex Curriculum & Book Assignment Flow
The current UI for assigning curricula and books to students is overly complex, requiring multiple steps and navigation across several pages. This slows down faculty and admin operations significantly.
✦ RECOMMENDED FIX
Redesign the assignment flow as a single-page wizard with bulk selection, drag-and-drop ordering, and a confirmation summary. Add a "Quick Assign" option for applying the same curriculum to multiple students at once.
